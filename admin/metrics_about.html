What are Object-Oriented Metrics?

Object-oriented (OO) metrics measure the quality, complexity, and maintainability of object-oriented software. They help developers identify potential issues, improve code design, and maintain healthy software architecture. The most widely used OO metrics were introduced by Chidamber and Kemerer (CK metrics) in 1994.

CK Metrics (Chidamber & Kemerer)

WMC - Weighted Methods per Class
Measures the complexity of a class based on the number and complexity of its methods. A high WMC indicates a class that may be doing too much and should be split.
WMC = Σ complexity(method_i) for all methods in class
Thresholds: > 20 methods = warning, > 50 methods = high risk
RFC - Response For a Class
Counts the number of methods that can be invoked in response to a message sent to an object of the class. It includes methods in the class and methods in other classes that are called.
RFC = M + R (M = methods in class, R = methods called by class)
Thresholds: > 50 = high response complexity
LCOM - Lack of Cohesion of Methods
Measures how closely the methods of a class are related to each other. High LCOM indicates methods are not well-related and the class may have multiple responsibilities.
LCOM = |P| - |Q| where P/Q are method pairs sharing attributes
Thresholds: > 0.8 = poor cohesion, > 0.5 = moderate issues
CBO - Coupling Between Objects
Counts the number of other classes to which a class is coupled. High coupling makes classes difficult to change and test.
CBO = number of classes that reference this class
Thresholds: > 5 = high coupling, > 10 = very high coupling
DIT - Depth of Inheritance Tree
Measures how deep a class is in the inheritance hierarchy. Deep inheritance can make code harder to understand and maintain.
DIT = number of ancestors
Thresholds: > 5 = deep hierarchy, > 2 = moderate depth
NOC - Number of Children
Counts the number of immediate subclasses of a class. Many children indicate the class is heavily extended.
NOC = number of immediate subclasses
Thresholds: > 10 = heavily extended base class

Additional Metrics

Number of Methods per Class
Simple count of methods in a class. Used as a basic complexity indicator.
Thresholds: > 20 = too many methods
Number of Attributes per Class
Count of instance variables (properties) in a class.
Thresholds: > 6 = too many attributes
Cyclomatic Complexity per Method
Measures the number of linearly independent paths through a method's source code. Higher complexity indicates harder to test and maintain code.
CC = E - N + 2P (E=edges, N=nodes, P=paths)
Thresholds: > 10 = complex method, > 20 = very complex
Lines of Code (LOC) per Class
Total number of lines of code in a class. Large classes are harder to understand.
Thresholds: > 300 LOC = very large class
